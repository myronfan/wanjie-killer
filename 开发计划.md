# 完结杀手 — 开发计划

> **核心愿景**：构建一个**自演化虚拟小镇**——星露谷物语风格的 2D 世界,
> 网文角色作为 Agent 在其中生活、互动、创作。世界通过 Build AI(LLM + 图像生成 +
> 代码生成)自主演化,无需人类干预即可无限发展。
>
> **基线**：V6.4.1 已完成(前端 React + Vite,后端 Spring Boot + MiniMax)。
> 本计划是基于 V6.4.1 的升级路线,采用三个开源参考项目的最佳实践。

借鉴 a16z 的 `useHistoricalValue` hook,解决"位置卡顿"问题：

```typescript
// 服务器每秒更新一次,但客户端按 60 fps 渲染
function useHistoricalValue(buffer: HistoryBuffer) {
  const [now, setNow] = useState(performance.now());
  
  useEffect(() => {
    let raf: number;
    const tick = (t: number) => {
      setNow(t);
      raf = requestAnimationFrame(tick);
    };
    raf = requestAnimationFrame(tick);
    return () => cancelAnimationFrame(raf);
  }, []);
  
  // 用 history buffer 平滑插值
  return interpolate(buffer, now);
}
```


#### 记忆系统(a16z 设计 + 我们增强)

```typescript
// 短期记忆：当前对话上下文(类似 LLM context)
// 长期记忆：对话结束后 LLM 总结 + embedding 存入向量数据库
// 反思：定期生成高层次的"经验总结"

class MemorySystem {
  // 对话结束后调用
  async rememberConversation(playerId, conversationId) {
    // 1. 加载对话消息
    const messages = await loadMessages(conversationId);
    
    // 2. LLM 总结对话
    const summary = await llm.summarize(
      `You are ${player.name}, summarize from your perspective: ${messages}`
    );
    
    // 3. 计算 embedding
    const embedding = await embedding(summary);
    
    // 4. 存入向量数据库,带 importance 分数
    await vectorDB.insert({
      playerId, summary, embedding,
      importance: await calculateImportance(summary),
      createdAt: Date.now()
    });
    
    // 5. 触发反思
    await this.reflect(playerId);
  }
  
  // 检索相关记忆(新对话开始时)
  async retrieveMemories(playerId, query, topK = 3) {
    const queryEmbedding = await embedding(query);
    return await vectorDB.search(queryEmbedding, topK);
  }
  
  // 反思：生成高层次经验
  async reflect(playerId) {
    const recentMemories = await vectorDB.getRecent(playerId, 100);
    const questions = await llm.generate(
      `Based on these recent memories, what are 3 high-level questions?`
    );
    for (const q of questions) {
      const relatedMemories = await this.retrieveMemories(playerId, q);
      const insight = await llm.generate(
        `Based on these memories: ${relatedMemories}, answer: ${q}`
      );
      await vectorDB.insert({
        playerId, summary: insight, type: 'reflection',
        importance: 8
      });
    }
  }
}
```

## 五、Build AI(核心创新)

> **这是本项目的核心创新点**：让 AI 角色自主创建新道具、新建筑、新规则。

### Build AI 工作流

```
NPC 想做新东西(由 LLM 决策触发)
  ↓
1. 需求分析(LLM)
   - 询问 NPC 想要什么
   - 输出：{ name, description, type }
  ↓
2. 类型分类(LLM)
   - 判定：item / building / recipe / rule / npc_recruit
   ↓
3. 配方设计(LLM → JSON)
   - 所需材料
   - 制作时间
   - 世界效果
  ↓
4. 图像生成(MiniMax 图像 API)
   - 生成 sprite PNG
   ↓
5. 代码生成(LLM)
   - 道具行为代码
   - 建筑交互代码
   ↓
6. Sandbox 测试
   - 语法检查
   - 静态分析
   - 运行时隔离
  ↓
7. 注册到世界
   - 写入数据库
   - 通知其他 NPC
```

### Build 权限分级

| 等级 | 角色 | 可生成 |
|------|------|--------|
| L1 | 村民 | 无(只能对话/移动/采集) |
| L2 | 工匠 | 同类(铁匠做武器,农夫做种子) |
| L3 | 大师 | 建筑/规则/剧情 |
| L4 | 创世者 | 修改世界本身(剧情触发) |

#### 1. 动画系统原理

```csharp
// StardewValley/AnimatedSprite.cs 核心
public class AnimatedSprite {
  public float timer;              // 计时器
  public float interval = 175f;    // 帧间隔(175ms ≈ 5.7 FPS)
  public int currentFrame;
  public int framesPerAnimation;   // 默认 4
  public Rectangle sourceRect;     // sprite sheet 切片
  public bool loop;
  
  // 核心更新
  public void Animate(GameTime time, ...) {
    timer += time.ElapsedGameTime.TotalMilliseconds;
    if (timer > interval) {
      currentFrame++;
      timer = 0f;
      if (currentFrame >= framesPerAnimation * 2 && loop)
        currentFrame = framesPerAnimation;
    }
    UpdateSourceRect();
  }
}
```

**应用到 PixiJS**：

```typescript
// 前端对应实现
class AnimatedSprite {
  timer = 0;
  interval = 175;  // ms — 与星露谷一致
  currentFrame = 0;
  framesPerAnimation = 4;
  
  update(deltaMs: number) {
    this.timer += deltaMs;
    if (this.timer >= this.interval) {
      this.currentFrame++;
      this.timer = 0;
      if (this.currentFrame >= this.framesPerAnimation * 2)
        this.currentFrame = this.framesPerAnimation;
    }
    this.updateSourceRect();
  }
}
```

## 十、安全与成本

### Sandbox 安全(LLM 生成代码必须经过)

**LLM 生成代码可能的风险**：
- 死循环阻塞世界
- 内存炸弹
- 调用不该调的 API

**三层防御**：
1. **语法检查**：AST 解析 → 拒绝语法错误
2. **静态分析**：检测危险调用(eval/Function/fs)
3. **运行时隔离**：QuickJS 或 GraalVM 沙箱

**人类审核模式**(可选)：
- L3/L4 高风险操作需要玩家点确认
- 在 `AutoPlayerToggle` 中可切换"全自动 / 半自动"

### LLM 成本控制

- **决策分级**：村长 30s/次,村民 5min/次
- **批量决策**：同区域 NPC 共享 1 次 LLM 调用
- **决策缓存**：相似情境缓存 1 小时
- **小模型降级**：简单 NPC 用本地模型,复杂用 MiniMax

### 记忆衰减

- 重要度 < 3 的记忆,30 天后归档
- 每 N 天 LLM 总结近期记忆成 1 段反思
- 冷数据归档到文件,热数据存 Redis/数据库

### 剧本 vs 自演化冲突

- **剧本 = 软引导**(只通过 system prompt 暗示)
- **AI = 硬自由**(完全自主)
- **冲突检测**：AI 偏离剧本 → 降级剧本到可选支线
